# 🚀 SyncScrap: Высокоэффективный асинхронный скрапинг-фреймворк

SyncScrap — это высокопроизводительный, распределённый фреймворк для скрапинга веб-страниц, построенный на базе Python. Он спроектирован для эффективного обхода сложных антибот-систем, извлечения данных и их обработки в реальном времени. Архитектура SyncScrap ориентирована на масштабируемость, отказоустойчивость и простоту развёртывания с использованием Docker.

## ✨ Основные возможности

- **Распределённая архитектура:** Микросервисная структура для обработки больших объёмов данных. 
- **Продвинутый Anti-Bot обход:** Использование `curl_cffi` для имитации реальных браузерных отпечатков (TLS/JA3/HTTP2), что минимизирует риски блокировок. 
- **Очереди сообщений:** Интеграция с RabbitMQ для надёжной передачи задач и результатов между сервисами.
- **Кэширование:** Использование Redis для кэширования HTML-страниц и ускорения работы.
- **Персистентность данных:** Хранение извлечённых данных в PostgreSQL с управлением миграциями через Alembic.
- **Асинхронность:** Полностью асинхронная обработка для максимальной производительности.

## 💡 Архитектура

Проект состоит из нескольких ключевых микросервисов, каждый из которых выполняет определённую роль:

- **`api/`**: Внешний API для взаимодействия с системой (добавление задач, просмотр результатов).
- **`worker_downloader/`**: Воркер, отвечающий за загрузку веб-страниц, включая обход антибот-систем и повторные попытки при ошибках.
- **`worker_parser/`**: Воркер для парсинга загруженных HTML-страниц и извлечения структурированных данных (на базе `selectolax`).
- **`worker_db/`**: Воркер для сохранения извлечённых данных в базу данных PostgreSQL.
- **`worker_scheduler/`**: Воркер, отвечающий за планирование задач скрапинга и отправку URL-ов в очередь загрузки.
- **`persistence/`**: Модуль для работы с базой данных (модели SQLAlchemy, DAO, миграции Alembic).
- **`shared/`**: Общие ресурсы и утилиты, используемые всеми сервисами (конфигурация, клиенты RabbitMQ/Redis, общие схемы).

Все сервисы разворачиваются и управляются с помощью Docker Compose.

## 🛠️ Стек технологий

- **Python 3.11+**
- **FastAPI** & **Uvicorn** для API
- **FastStream** для очередей сообщений (RabbitMQ)
- **SQLAlchemy** & **Asyncpg** для ORM и асинхронной работы с PostgreSQL
- **Alembic** для миграций базы данных
- **Redis** для кэширования
- **curl_cffi** для HTTP-запросов с обходом антибот-систем
- **Selectolax** для быстрого парсинга HTML
- **Docker** & **Docker Compose** для контейнеризации и оркестрации

## 🚀 Запуск проекта

Для локального запуска проекта вам потребуется Docker и Docker Compose.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <ваш_репозиторий>
    cd SyncScrap
    ```

2.  **Настройте переменные окружения:**
    Создайте файл `.env` в корне проекта (рядом с `docker-compose.yaml`) и заполните его необходимыми переменными для подключения к БД, RabbitMQ, Redis. Пример:
    ```ini
    DB_USER=user
    DB_PASSWORD=password
    DB_NAME=syncscrap_db
    RABBITMQ_USER=guest
    RABBITMQ_PASS=guest
    REDIS_HOST=redis
    DOWNLOADER_HTTP_IMPERSONATE=chrome131
    DOWNLOADER_HTTP_TIMEOUT_SECONDS=60.0
    DOWNLOADER_HTTP_MAX_CLIENTS=10
    ```

3.  **Запустите сервисы:**
    ```bash
    docker compose up -d --build
    ```

4.  **Проверка:**
    После запуска все сервисы должны быть подняты и готовы к работе. Вы можете проверить их статус с помощью `docker compose ps`.
